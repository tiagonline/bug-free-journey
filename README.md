# Desafio Técnico - QA Lead (Sigga)
**Autor:** Tiago Oliveira Silva

## Introdução

Este repositório contém os itens do "Desafio Técnico - QA Lead".

A estrutura deste projeto foi organizado em duas áreas principais:

1. **/automacao**: Contém o código-fonte funcional dos projetos de automação (Cypress e K6).
2. **/entregaveis**: Contém toda a documentação estratégica e a análise de métricas, separada por entregável.

## 1. Plano Estratégico de Testes (Alto Nível)

Fiz a análise de arquitetura, pontos de falha, tipos de teste, riscos e os 5 cenários críticos  que considero obrigatório para uma primeira bateria de testes antes do go-live.
* **Arquivo:** [**`Ver Entregável 1`**](./entregaveis/entregavel_1.md)

## 2. Automação de Cenários Essenciais

Justifico a escolha de cada *frameworks* (Cypress e K6). O código-fonte está na pasta `/automacao`.
* **Arquivo:** [**`Ver Entregável 2`**](./entregaveis/entregavel_2.md)

## 3. Métricas e Análise de Qualidade

Minha proposta de indicadores (KPIs), o relatório simulado e a análise de métricas acionáveis.
* **Arquivo:** [**`Ver Entregável 3`**](./entregaveis/entregavel_3.md)

## 4. Desafios Adicionais (Escala e Privacidade)

Fiz a estratégia para testes geolocalizados, paralelos e validação de privacidade (LGPD/GDPR).
* **Arquivo:** [**`Ver Entregável 4`**](./entregaveis/entregavel_4.md)
